NAIMI-TREHEL's ALGORITHM *DISTRIBUTED COMPUTING IN A GRID*

What the heck is Naimi Trehel's algorithm? And how is it used in a distributed grid network?
 

Foreplay

Naimi-Trehel's algorithm
Second algorithm in the token based protocols for synchronizing resource sharing in a grid. The introduction of the syncrhonization and distributed computing in a grid is covered here.
  • Naimi-Trehel's algorithm achieves the synchronized resource sharing in the grid by maintaining just 2 structures. One is a logical dynamic tree and a queue. 
  • A dynamic tree is a data structure where the root of the tree changes every now and then. 
  • The author of this book calls these both LAST tree and NEXT queue respectively.
  • Each node holds 2 references. One points to the root node of the LAST tree and the NEXT node i.e which node should receive the token next w.r.t it.

The main part!

So how do we determine what will be the root of the LAST tree and what will be the NEXT node? The root of the LAST tree is a node which will receive the token last. The NEXT node w.r.t to a node is the node which should receive the token next. 


Initially all the nodes point their LAST reference to the the node which holds the token. Now when a node requests for the token , the initial LAST node will point its LAST and NEXT pointers to this requested node. Similarly , when another node comes in with a request for the token, these 2 nodes point their LAST to the node which requested the token.

For example, Consider 5 nodes A, B, C, D, E where A holds the token. So first step is , all the other nodes point their LAST to A. Now, C comes in with a request to the token. A's LAST and NEXT pointers are updated and pointed to C. Then E comes in with a token request , now C's and A's  LAST pointer is updated to point towards E and C's NEXT is updated to E. And then D comes in with a  token request , E's, C's, A's LAST now point to D and E's NEXT will point to D.

After all this mapping and pointing is done, the token is released from the first LAST node , and it moves in the direction NEXT. It follows the NEXT pointer of each node, traversing through all the nodes in the NEXT queue { When a node comes in with a token request, it is automatially added to the NEXT queue when the NEXT pointer of the previous node is updated to this node } and finally reaches the root node of the LAST tree which is C in the given example and one token circulation cycle ends there. Now when the token enters a node , that node will enter its critical section and after that is done, it sends the token to the NEXT node w.r.t to it.

Climax

That should conclude Naimi-Trehel's algorithm.  Note that this series is a short notes on some protocols from "Fundamentals of grid computing".
