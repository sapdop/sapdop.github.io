PATTERNS - BACKTRACKING

How to generate random patterns with N number of tiles in an MxM grid for tile based puzzles using basic data structures like stacks.
 

Foreplay

random patterns
When you're starting out with writing your own tile based game , one of the major hurdles would be creating the levels for it. An algorithm which generates endless patterns by taking in number of tiles needed would solve this problem big time! 
  • So, the magical algorithm here would be something that just takes in the number of tiles required for a level , and generates a random pattern in a given grid with N tiles.
  • A modified version of this algorithm is used in one of my games "I'm not racist". The mechanic of the game is pretty simple. You have X number of color tiles in each level. Then there are Y number of races of people. Player's goal is to make sure that no two people of same race end up under same color tile group. Each level will have N number of tiles and they are divided into seperate color tiles.

The main part!

Pattern Generation - Backtracking

Take a stack. Select a random cell in the grid. Push that cell into the stack and place a tile there. Now do a random walk to any of the adjacent cells . Then push that cell to the stack. Keep on doing a random walk till you reach a dead-end. 

Dead-end case : There are no adjacent cells available to the current cell.

In the event of a dead-end , Pop the current cell out of the stack so that the previous cell comes to the Top of the stack. Now use the previous cell as the current cell and check if there are any available adjacent cells to it. Let this algorithm dance till you have the required number of tiles in place and then stop.

Climax

That's it! This is a pretty beginner algorithm that lets you generate random patterns. To this skeleton you can put in stuff to generate unique pattens or something that suits your game mechanic.

Peace out!
