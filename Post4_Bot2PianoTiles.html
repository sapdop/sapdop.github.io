BOTS PLAY GAMES #2
"PIANO TILES"

Second part of the series where we try and breakdown some games and write individual bots to play them.
 

Foreplay

Please don't say that you've never played Piano Tiles before!

Piano Tiles is a mobile game with a linear gameplay tailored into different game modes for variation. There is nothing much to talk about the art-style, flat colors everywhere but the level of polishing in the gameplay is the key here when you aim to make a game with minimal art. You can download the game for free here for your android emulator. 

  • Well, many people have already wrote bots for this particular game and espcially one guy who did an amazing job at making an actual player simulation using 4 mechanical arms and the bot code running in the background, instructing these arms to fire when needed. You can watch that video here.
  • But this is another simple game to breakdown. Honestly , I couldn't find any better game after a rough day, so please deal with it.
  • Alright , the gameplay is pretty simple. There are 4 columns and you're supposed to tap on the correct tile for each row ( the black tile) . So if you already implemented the bot from the last episode , this will be a piece of cake. 

The main part!

BOT PIPELINE

So this is basically how the bot works , and you're the wizard who'll be writing that magic part.

The bot pipeline stays the same , you get some input , the bot does the magic and gives some output.

Input : Well, for input, we need pixel data from 4 different regions. Because for each row , we check which column cell is black and tap on that cell. So, we get the data for one pixel in each of these points of action.

Magic : For each input cycle , you get the data of 4 pixels (You can get away with obtaining just the R value from RGB). Now, the checking can be done with 4 if conditions. If the R value of any cell falls under the black threshold , tap on that cell. As black and white sit on either ends of the spectrum , differentiating the colors won't even bother you in this case. 

Output : Now how will you tap? If you're using an emulator, you'll probably have the input mapping option , so you can do it this way where you map a particular key press to a tap at a particular location. Or the other way where you somehow move the mouse pointer to a particular location each time and click  (I've personally never used any library that does this in python, If you do so please let me know).

Note : Same as the last one. There is a subtle time delay for a change in row of tiles after you successfully tap one. So make sure you have a wait time triggering after every iteration. And I'm not sure about this but the pace of the game increases with your score I guess. So if that is the case, make sure that you're wait time value varies depending on the pace of the game.

So, get the R values from any pixels in all the 4 points of action (technically they're regions) and feed it to your magical code.

Climax

Now you have 2 bots in your arsenal. Good job! I remember OpenAI working on a bot platform where the bots learn to play and master games with some reinforcement learning. Do check it out. That's it for the day I guess.
