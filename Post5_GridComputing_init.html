SYNCHRONIZED RESOURCE SHARING  *DISTRIBUTED COMPUTING IN A GRID*

How do those pool of computers with shared resources manage synchronization?
 

Foreplay

distributed computing in a grid
Consider an ATM. You have an account in a particular bank but that doesn't stop you from withdrawing cash from any other ATM available. How does that work? Well, picture all these ATMs as nodes in a huge network capable of functioning on their own. But they all share something, resource of knowledge. Probably your card might have the details like which bank's database to redirect to and your account details , and if that is in this individual ATM's scope , it fetches stuff and lets you access the details and withdraw cash unless you're broke like me. This is in one context where they call this magic - distributed computing. 
  • Distributed computing in a grid network , has many nodes but mostly all of them share some common resources because they  work together. So it is essential for them to be synchronized.
  • Ever heard of critical section? I'll put it this way. You open a file on your computer , for example a video file. Play it , and while playing try to delete the file. It obviously shows you a window saying "It's not possible , some other program is using the file." right? Now why does that happen? Your operating system is designed in such a way that , when a process is using a shared resource, no other process can access that resource or operate on it. This whole thing of operating on a shared resource is called  "critical section" of a process. It is a segment of code where the process operates on a shared resource. 
  • Similarly, when you're dealing with an arsenal of nodes with some shared resources , you obviously have to take care of scheduling the critical section operations of all the nodes for all of them to be synchronized and to avoid collisions.

The main part!

Types of syncrhonization protocols : There are many protocols out there which help you with this synchronization. They all can be classified mainly into 2 types. 

        1) Permission based protocols : For a node in the distributed network to enter into its critical section, it has to obtain permission from all the other nodes.  

        2) Token based mutex protocols : A token is circulated across the distributed network. For a node to enter into its critical section , it has to have the token. (Mutex - preventing simultaneous access to a shared resource).

Climax

That's it for today's short post on what distributed computing in a grid is. Looks like most of it is just foreplay. Permission based algorithms like Lamport and Maekawa , token based algorithms like Martin , Naimi-Trehel and Suzuki Kasami will be covered in the future posts. Stay tuned!
